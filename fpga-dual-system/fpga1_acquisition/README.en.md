# FPGA1 Acquisition

## Purpose

This folder contains the FPGA1-side acquisition and transmit subsystem.
Its job is to take a sample source selected at board-wrapper level, prepare
the sample for local display and status reporting, package the result into a
fixed UART frame, and transmit that frame to FPGA2.

## Folder Layout

### `src/`

Synthesizable RTL used by the FPGA1 acquisition path.

### `src/pkg/`

- `fpga1_pkg.vhd`: FPGA1-specific constants and helper functions. It centralizes
  default parameters such as ADC timing, display scaling, fake-sensor defaults,
  and helper conversion from sample code to display temperature.

### `src/processing/`

- `fake_sensor_gen.vhd`: Synthetic sample generator used for simulation,
  bring-up, and wrapper-level test modes. It can generate ramping or pseudo-
  random sample codes and can also force a known invalid code for tests.
- `sample_classifier.vhd`: Converts the current validity/range information into
  a sensor-state code used later by the frame builder.
- `sample_noise_monitor.vhd`: Sliding-window spread monitor intended to detect
  EMI-like noise on incoming samples. In the current tree it is an auxiliary
  block that exists for integration/testing work.
- `sample_normalizer.vhd`: Converts the incoming raw sample code into a bounded
  display temperature value used by the local decimal display path.
- `sample_validator.vhd`: Checks whether a sample is inside the accepted raw
  range and produces `range_ok` / `range_error`.
- `threshold_detector.vhd`: Standalone helper that derives warning and error
  flags directly from a sample value. It is useful for diagnostics or alternate
  wiring, but it is not the main framing path in the current top-level.

### `src/display/`

- `bin_to_bcd.vhd`: Converts a binary display value into BCD digits so the
  number can be shown on the seven-segment displays.
- `digit_to_7seg_decimal_n.vhd`: Converts one decimal digit into the active-low
  seven-segment pattern used by the DE10-Lite displays.

### `src/link_tx/`

- `frame_builder.vhd`: Builds the outgoing fixed 8-byte UART frame
  (header/control/sequence/sample/flags/CRC/footer).
- `uart_tx.vhd`: Byte-oriented UART transmitter used to send the frame to
  FPGA2.

### `src/top/`

- `fpga1_top.vhd`: Main reusable FPGA1 core. It receives prepared samples from
  the board wrapper, drives local status/display logic, builds UART frames, and
  transmits them.

### `sim/`

Simulation-only helpers and testbenches.

- `uart_rx_monitor.vhd`: Lightweight UART receiver used by FPGA1 testbenches to
  observe transmitted bytes.
- `tb_fake_sensor_gen.vhd`: Unit-style testbench for the fake sensor generator.
- `tb_fpga1_top.vhd`: Main FPGA1 top-level testbench. It drives sample inputs,
  checks display output, and validates the generated UART frame.

### `quartus/`

Project setup files for building FPGA1 in Quartus.

- `add_sources.tcl`: Adds the reusable FPGA1 source files to a Quartus project.
- `add_de10_lite_wrapper_sources.tcl`: Adds FPGA1 sources plus board-wrapper
  and DE10-Lite specific files.
- `de10_lite_placeholder.qsf`: Minimal placeholder Quartus settings file used
  as a starting point.
- `de10_lite_fpga1_wrapper.qpf`: Quartus project file for the DE10-Lite FPGA1
  wrapper build.
- `de10_lite_fpga1_wrapper.qsf`: Quartus assignments file listing sources and
  project settings for the FPGA1 wrapper target.
- `de10_lite_fpga1_wrapper.sdc`: Timing constraints for the FPGA1 wrapper
  project.
- `unsaved/unsaved_generation.rpt`: Temporary Quartus-generated report artifact.
- `.qsys_edit/`: Quartus editor state generated by the tool.
- `db/`: Quartus compilation database generated by the tool.
- `incremental_db/`: Quartus incremental compilation data generated by the
  tool.
- `output_files/`: Generated output reports and build artifacts from Quartus.

## Workflow

1. A board-level wrapper outside this folder selects the actual source of the
   sample: real ADC, fixed test source, or `fake_sensor_gen`.
2. That wrapper feeds `sample_value_i` and `sample_valid_i` into
   `src/top/fpga1_top.vhd`.
3. `sample_normalizer.vhd` prepares a user-friendly temperature value for the
   local seven-segment display path.
4. `sample_validator.vhd` checks whether the raw sample is inside the accepted
   operating range.
5. `sample_classifier.vhd` produces the sensor-state code used by the outgoing
   frame.
6. `fpga1_top.vhd` latches the current sample/status, updates the local display
   outputs and local error indicator, and requests a new frame.
7. `frame_builder.vhd` assembles the fixed UART frame and computes the CRC.
8. `uart_tx.vhd` serializes each byte onto the inter-FPGA UART link.

## Practical Use

- Use `src/` when changing synthesizable FPGA1 behavior.
- Use `sim/` when validating RTL changes with GHDL or another simulator.
- Use `quartus/` when preparing or rebuilding the DE10-Lite FPGA1 project.
- Treat `quartus/db/`, `quartus/incremental_db/`, `quartus/output_files/`, and
  similar tool-generated folders as build state, not as design source.
