# FPGA2 Display

## Purpose

This folder contains the FPGA2-side receive, link-monitoring, and display
subsystem. Its role is to receive the UART stream coming from FPGA1, decode and
verify each frame, track communication health, and present the received system
state on local display outputs.

## Folder Layout

### `src/`

Synthesizable RTL used by the FPGA2 receive and display path.

### `src/pkg/`

- `fpga2_pkg.vhd`: FPGA2-specific constants, mainly LED patterns used by the
  optional LED status path.

### `src/link_rx/`

- `uart_rx.vhd`: Byte-oriented UART receiver that samples the incoming serial
  line and emits received bytes with a valid pulse.
- `frame_decoder.vhd`: Reassembles the fixed 8-byte frame, checks header,
  control field, footer, and CRC, and emits decoded sample/status data plus
  valid/corrupt activity pulses.
- `internal_uart_frame_gen.vhd`: Optional internal UART frame generator used
  for self-test or simulation when FPGA2 needs to run without a real FPGA1
  transmitter.

### `src/control/`

- `link_statistics.vhd`: Tracks total frames, valid frames, corrupted frames,
  missing frames, timeout events, current sensor value, current sensor state,
  and overall communication state.
- `status_mapper.vhd`: Maps sensor state and communication state into a 4-bit
  LED pattern. It is useful as a support block for LED-based status displays.

### `src/display/`

- `led_driver.vhd`: Registers and holds the current LED pattern cleanly.
- `vga_timing.vhd`: Generates pixel enable, active video, pixel coordinates,
  and horizontal/vertical sync signals for the VGA pipeline.
- `vga_dashboard.vhd`: Renders the FPGA2 dashboard content: current sensor
  value, link counters, communication state, sensor-state indicators, and the
  vertical gauge.

### `src/top/`

- `fpga2_top.vhd`: Main reusable FPGA2 core. It selects the UART source,
  receives and decodes frames, updates link statistics, drives VGA output, and
  controls the board-facing status displays.

### `sim/`

Simulation-only files and testbenches.

- `tb_link_statistics.vhd`: Unit-style testbench focused on the statistics and
  timeout logic.
- `tb_fpga2_top.vhd`: Main top-level FPGA2 testbench using an external UART
  stimulus.
- `tb_fpga2_internal_source.vhd`: FPGA2 testbench that enables the internal
  UART frame generator and checks self-test behavior without FPGA1.

### `quartus/`

Project setup files for building FPGA2 in Quartus.

- `add_sources.tcl`: Adds the reusable FPGA2 source files to a Quartus project.
- `add_de10_lite_wrapper_sources.tcl`: Adds FPGA2 sources plus board-wrapper
  specific files.
- `de10_lite_placeholder.qsf`: Minimal placeholder Quartus settings file used
  as a starting point.
- `de10_lite_fpga2_wrapper.qpf`: Quartus project file for the DE10-Lite FPGA2
  wrapper build.
- `de10_lite_fpga2_wrapper.qsf`: Quartus assignments file listing sources and
  settings for the FPGA2 wrapper target.
- `de10_lite_fpga2_wrapper.sdc`: Timing constraints for the FPGA2 wrapper
  project.
- `db/`: Quartus compilation database generated by the tool.
- `incremental_db/`: Quartus incremental compilation data generated by the
  tool.
- `output_files/`: Generated reports and output artifacts from Quartus.

## Workflow

1. `fpga2_top.vhd` selects the UART source:
   the real board input or the optional `internal_uart_frame_gen.vhd`.
2. `uart_rx.vhd` converts the incoming serial stream into bytes.
3. `frame_decoder.vhd` rebuilds the fixed UART frame and verifies its
   structure and CRC.
4. `link_statistics.vhd` accumulates communication counters and keeps the most
   recent valid sensor value/state plus the current communication state.
5. `vga_timing.vhd` generates the timing and pixel coordinates for the VGA
   raster.
6. `vga_dashboard.vhd` turns the current statistics and sensor data into a live
   VGA dashboard.
7. `fpga2_top.vhd` also maps communication state to the local seven-segment
   board display so the board can show states such as `GOOD`, `NONE`, or
   `ERROR`.

## Notes About Support Blocks

- `status_mapper.vhd` and `led_driver.vhd` are useful support modules for a
  dedicated LED status path.
- In the current top-level tree, FPGA2 keeps board LEDs forced off and uses the
  seven-segment display plus VGA as the active user-facing outputs.

## Practical Use

- Use `src/` when changing synthesizable FPGA2 behavior.
- Use `sim/` when validating receive-path and display-path changes in
  simulation.
- Use `quartus/` when preparing or rebuilding the DE10-Lite FPGA2 project.
- Treat `quartus/db/`, `quartus/incremental_db/`, and `quartus/output_files/`
  as generated build state rather than hand-maintained source.
